# 부서별 업무링크 정적 사이트 버그 수정 보고서

## 1. 개요 (Context & Purpose)
사용자가 교사 로그인(`Master` / `2026`) 상태에서 업무 카드를 신규 추가하거나 수정할 때 발생하는 두 가지 오류 현상을 해결하고자 합니다.
- **현상 1**: 구글 스프레드시트 4열(parentId)에 한글 부서명이 아닌 영문 부서 ID(예: `dept-admin`)가 기입되는 현상
- **현상 2**: 마인드맵 보드에서 2단계 업무카드 링크의 연결선(점선 화살표 엣지)이 여러 겹 겹쳐서 소용돌이 형태 등으로 중복 생성되는 현상

---

## 2. 원인 분석 (Rationale)

### 2.1 4열 영문 부서 ID 저장 오류
- **분석 결과**: 클라이언트 상태 및 화면 마인드맵 렌더링을 위해서는 부서 노드의 고유 ID(영문, 예: `dept-admin`, `dept-career`)가 링크 카드의 `parentId`로 사용되어야 정상적으로 엣지가 그려집니다. 그러나 구글 스프레드시트와의 동기화 및 저장 규약상 **스프레드시트 4열에는 한글 부서명**이 기입되도록 설계되었습니다.
- 실제로 데이터를 로드하는 시점(`src/app/page.tsx`)에는 이미 4열의 한글 부서명을 영문 ID로 역치환하여 클라이언트에 주입해주는 로직이 구현되어 있었습니다.
- 그러나 저장(POST)을 수행하는 `syncAllCardsToSheets` 함수(`src/lib/card-data.ts`) 내에서 `parentId`를 한글명으로 바꿔서 전송해주는 매핑 처리가 누락되어 영문 부서 ID가 그대로 구글 스프레드시트에 기입되고 있었습니다.

### 2.2 연결선 중복 렌더링 오류
- **분석 결과**: 구글 스프레드시트 또는 API 동기화 과정에서 카드 데이터가 중복하여 누적 생성되거나, 비동기 호출 시 동일한 고유 ID(`id`)를 가진 데이터가 여러 번 로드되는 현상이 존재했습니다.
- 클라이언트가 이 데이터를 받아 그대로 상태(`cards`)로 인스턴스화할 때 `unique` 처리가 되어있지 않아, 동일한 노드가 `cards` 배열 내에 여러 개 복사되었습니다.
- SVG 연결선을 그리는 루프(`MindMapBoard.tsx`)에서 중복된 개수만큼 패스(점선) 및 마커 화살표가 생성되어 한 곳에 여러 번 겹쳐 그려지면서 화살표 무리가 겹쳐서 렌더링되었습니다.

## 6. 추가 기능 구현: 긴 텍스트의 단어 단위 가변 줄바꿈 적용 (Word Break Keep-All)
- **요구사항**: '2028 대학입학전형 시행계획 검색기'와 같이 업무카드명이 긴 경우, 끝 글자 한 두자만 단독으로 다음 줄에 덜렁 개행(`검색\n기`)되어 가독성을 저해하는 문제를 가변적인 단어 단위 줄바꿈으로 개선하고자 합니다.
- **조치 내용**:
  - `DepartmentCard.tsx`의 2단계 업무카드 텍스트를 표현하는 `span` 태그에 Tailwind CSS의 `break-keep` (`word-break: keep-all;`) 클래스를 추가했습니다.
  - 이로써 CJK 한글 텍스트 렌더링 시 브라우저가 어절(공백) 단위로만 줄바꿈을 수행하여, 단어 중간에 글자가 끊기지 않고 `'2028 대학입학전형' \n '시행계획 검색기'` 와 같이 읽기 편한 구조로 개행이 자연스럽게 적용됩니다.

## 7. 추가 기능 구현: 업무 카드의 CJK 개행 모방 예측 너비 주입 (Width: 80px~190px 최적화)
- **요구사항**: 긴 글자의 카드가 줄바꿈되었을 때, 우측에 하얀 여백이 어색하게 남는 것을 없애고, 카드의 가로 크기가 전체적으로 실제 글자 단어 길이에 맞춰 타이트해지도록 개선하되, '광양고 홈페이지'나 '아이모두 전남'처럼 짧은 카드들은 줄바꿈 없이 한 줄로 깔끔하게 렌더링되기를 요청받았습니다.
- **조치 내용**:
  - 브라우저가 CJK 텍스트를 `keep-all`로 개행했을 때, 개행 후 텍스트 너비가 아닌 개행 전 너비에 가깝게 박스 가로폭을 크게 잡는 레이아웃 문제를 해결하기 위해 `grid.ts`의 `getLinkWidth` 함수에 개행 예측 픽셀 실측 알고리즘을 도입했습니다.
  - 가로 폭의 컴팩트함을 유지하면서 짧은 텍스트(예: 광양고 홈페이지, 아이모두 전남)가 강제로 줄바꿈되는 현상을 막기 위해, 링크 카드의 최대 가로 너비(`maxWidth`) 한도를 기존 `160px`에서 **`190px`**로 상향 조정했습니다.
  - 이로써 짧은 카드들은 줄바꿈 없이 한 줄로 온전히 표시되면서 글자 길이에 맞게 타이트해지고, 190px을 넘는 극도로 긴 카드들만 자연스럽게 두 줄 개행된 채 최장 라인 너비에 딱 맞춰 가로폭이 좁혀집니다.

## 8. 추가 기능 구현: 로그인 시 업무 카드 글자와 우측 편집/삭제 아이콘 간 빈 공간(여백) 제거
- **요구사항**: 교사 로그인 시 업무 카드의 텍스트 문자열과 우측 편집(연필) 및 삭제(휴지통) 아이콘 사이에 불필요하게 넓은 흰색 빈 공간(여백)이 생기는 현상을 해결하고 서로 바짝 밀착되도록 조정합니다.
- **조치 내용**:
  - `DepartmentCard.tsx` 내부 2단계 링크 카드의 텍스트를 감싸는 `span` 태그에 걸려 있던 **`flex-1`** 클래스를 제거하고 **`w-auto flex-none`**을 부여했습니다. 이로 인해 텍스트 영역이 우측 끝까지 불필요하게 늘어나는 것을 차단했습니다.
  - `grid.ts`의 `getLinkWidth` 함수에서 로그인 시 추가적으로 산입하던 아이콘 배치 공간인 `baseSpacing` 값을 현실적인 아이콘 세트 크기에 맞추어 `44px` ➔ **`32px`**로, 로그아웃 시 또한 `24px` ➔ **`18px`**로 대폭 단축하여 계산식 픽셀의 오차를 없앴습니다.
  - 더불어 글자 한 자당 기본 가중 픽셀 수치를 `11.2px` ➔ **`10.5px`**로 정밀 보정하여, 텍스트가 줄바꿈되었든 한 줄이든 관계없이 글자와 아이콘이 부모 카드 내부에서 딱 맞게 밀착하도록 조치했습니다.

## 9. 추가 기능 구현: 12자 초과 시 전체 글자수 기준 균형 있는 2줄 개행 및 스페이스 한 칸의 여유 공간 보장
- **요구사항**: 
  - 이전 `flex-none` 상태에서 발생했던 텍스트가 카드 밖으로 튀어나가고 수정/삭제 아이콘이 밀려나는 레이아웃 에러를 해결합니다.
  - 글자 수가 공백 포함 12자를 초과하면 두 줄로 자연스럽게 개행하되, 단어 한 개만 밑으로 덜렁 내려가지 않고 앞뒤 줄의 가로폭 밸런스를 맞춰 이분할 개행되도록 합니다.
  - 글자 끝 부분과 아이콘 영역 사이에 스페이스 한 칸(약 6px) 크기의 최소 안전 여백을 둡니다.
- **조치 내용**:
  - `DepartmentCard.tsx` 내부 텍스트 `span` 태그에 `flex-1 min-w-0 pr-1.5` 속성을 부여했습니다. `flex-1 min-w-0`를 통해 텍스트가 카드의 고정 너비 내에서 안전하게 줄바꿈되도록 롤백시켰고, `pr-1.5` 패딩을 통해 글자 끝과 아이콘 사이에 6px(스페이스 한 칸)의 시각적 여유 마진을 보장했습니다.
  - `grid.ts`의 `getLinkWidth` 알고리즘에 **균형 분할 밸런싱 로직**을 탑재했습니다. 단어 수 2개 이상의 긴 카드명에 대해, 단어 경계 분할점 중 앞줄과 뒷줄의 너비 편차가 최소화되는 분할 인덱스(Split Index)를 수학적으로 찾아내어 2줄 개행을 유도하고 최장 라인 길이에 맞추어 카드 박스의 실제 너비를 지정합니다.
  - 이에 따라 `'2028 대학입학전형 시행계획 검색기'`(21자)는 앞뒤 너비 밸런스를 고려하여 `'2028 대학입학전형'` \n `'시행계획 검색기'` 로 아주 균등하게 쪼개지며, 우측 여백이 메꿔진 채 안전하게 박스 내부에 들어차게 됩니다.
